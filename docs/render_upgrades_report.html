Vulkan post-process chain (locked order, best results): composite → scale (SGSR/FSR) → FXAA → Toon → Color → CAS → HDR → NTSC → CRT → screen. Anti-alias first, stylize, colour-grade, sharpen, bloom — then the output-medium emulation (NTSC signal, then CRT tube) runs last. Effects are independent composable toggles; each is optional and the chain skips what's off.

The effect passes chain after the upscaler (composite → scale → CAS → HDR → screen) with no extra GPU barriers.

Device test — do the upscalers actually do anything?

On-device A/B VERIFIED 2026-06-27

Real hardware (root bridge) · build 1.9.1 · 720p container → 1080p panel · frozen-frame comparison (game paused, compositor keeps re-upscaling)

A first test on a smooth raymarched SDF blob showed almost no difference between modes (<0.4% RMSE, black diff-maps). That was bad test content, not a bug — a smooth gradient has no high-frequency edges for an edge-reconstruction upscaler to work on. Re-tested on the DX11 "space" scene (textured planet + coastlines + dense starfield) the modes become clearly & usefully distinct:

Scaling modeRMSE vs NoneWhat you see on screen
Nearest0% (≡ None)hard stair-step jaggies on the planet's curved edge (point sampling)
Linear1.79%jaggies smoothed, but the whole frame softened / blurred (stars dimmer)
SGSR1.75%edges cleaned up while stars & detail stay crisp — the sweet spot
FSR / FSR-Fit1.82%same family as SGSR — reconstructed edges, retained detail
Sharpen2.46%noticeably brighter / punchier, more contrast (RCAS); keeps the base jaggies
Takeaways: (1) the upscalers work exactly as designed — these are spatial upscalers (not AI/temporal), so the benefit is an edge-cleanup that grows with the upscale ratio, ~6× stronger here than on the smooth scene. (2) "More RMSE" is not "better" — fidelity to a native render is the goal, and SGSR/FSR landing close to bilinear while keeping detail is the correct result. (3) For a more dramatic win, render at a lower internal res (e.g. 540p→1080p). (4) The on-device run also confirmed the Sharpen fix and is the reason the P1c sharpness sliders were added — the strength was previously locked at 0.25.
